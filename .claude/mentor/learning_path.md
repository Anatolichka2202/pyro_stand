## Уровень 2 — Ближайшие цели (1-2 месяца)

### C++/Qt

**A. Написать тест самостоятельно**
Задача: `tst_reading::delayedT0_eventFiresAfterKP()`
- MockSerial.fireAt(timeToStartMs + 10000, SYNC_MASK) — КП опаздывает на 10 сек
- MockSerial.fireAt(timeToStartMs + 12000, 0x03) — событие после КП
- Проверить: `events[0].firedTick == 12000`, `events[0].status != "fail"`
Это научит: как устроены тесты на читающий поток, как Mock работает внутри.

**B. `std::atomic` — практика**
Написать минимальный пример: два потока, один пишет, другой читает.
Сравнить поведение с `bool` и с `atomic<bool>` (под asan/tsan).

**C. Сигналы с QueuedConnection — практика**
Добавить `qDebug() << QThread::currentThreadId()` в слот `eventFired`.
Убедиться что ID отличается от ID в emit.

### Архитектура

**D. Прочитать главу про State Machine в GoF или аналоге**
В коде: `Phase::Idle → Loaded → Countdown → Running → Completed/Stopped`.
Это классическая State Machine. Читать с пониманием паттерна, а не просто enum.

**E. Добавить retry к команде СТОП**
`stop()` сейчас = один UDP-пакет. Добавить: 3 попытки × 100 мс задержка.
Это первая самостоятельная доработка бизнес-логики.

### UI/UX

**F. Проверить контрастность своих цветов**
Инструмент: https://webaim.org/resources/contrastchecker/
Взять 3 пары из кода (текст/фон) и проверить.

**G. Добавить прогресс-бар TFTP**
Stand уже знает количество блоков (DATA каждые 512 байт).
Добавить сигнал `tftpProgress(int percent)` и подключить к QProgressBar.
