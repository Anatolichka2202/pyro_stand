## Критические проблемы ✗

### 1. Сценарий задержки T0 не покрыт тестами

Баги из последней сессии (КП = НЕ СРАБОТАЛО, пустая таблица во время опоздавшего полёта)
были найдены только по логам с реального железа. Тесты не поймали.

**Что добавить:**
```cpp
// tst_reading.cpp
void delayedT0_tableShowsTick()
{
    // absoluteIndex > timeToStartMs до срабатывания КП
    // → eventFired должен эмититься из late-detection ветки
    // → firedTick в EventRow должен быть != -1
}

void delayedT0_analysisStatusIsLate()
{
    // После analyzeEvents: status=="late", deviationMs > 0
}
```

### 2. `eventFailed` сигнал объявлен, но не эмитируется

`stand.h:70` — `void eventFailed(int eventId);` — мёртвый код.
Если MainWindow когда-нибудь подключится к нему — ничего не придёт.
Либо удалить, либо эмитировать при реальном провале канала.

### 3. Кластеризация таймлайна игнорирует статус "late"

`timeline_widget.cpp` — в коде слияния кластеров:
```cpp
if (te.status == "fail") last.status = "fail";
else if (last.status != "fail" && te.status == "ok") last.status = "ok";
// "late" не обрабатывается → кластер остаётся серым
```

### 4. Команда СТОП без подтверждения

`stop()` отправляет UDP `0x63` один раз. Нет retry, нет ACK.
В боевых условиях — неприемлемо для команды останова.

## Некритические замечания ⚠

### 5. `m_masks` растёт неограниченно

`struct MaskRecord` записывается для каждого байта. Полёт 30 мин = 1.8M записей ≈ 16 МБ.
Не критично для современного PC, но стоит ограничить размер или не хранить вообще
(данные уже в лог-файле).

### 6. Прогресс TFTP не отображается

Пользователь не видит ход передачи циклограммы. При медленной сети —
кнопка "Загрузить" просто висит замороженной.

### 7. Несоответствие FLIGHT_SAFETY_MS в коде vs документации

`CLAUDE.md` говорит 1000 мс, `stand.h` содержит `FLIGHT_SAFETY_MS = 5000`.
Одно из двух устарело.

### 8. Валидация порядка событий циклограммы отсутствует

Если `cyclogram.ini` содержит события не в хронологическом порядке —
анализ может дать неверные результаты. Нет проверки `std::is_sorted`.
