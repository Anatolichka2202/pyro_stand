# Задачи для оркестратора

Эта папка — очередь задач. Оркестратор кладёт файл задачи сюда, AI-агент его забирает.

## Формат файла задачи

Имя файла: `TASK-NNN-короткое-описание.md`

```markdown
# TASK-001: Краткий заголовок

## Контекст
Что и почему нужно сделать. Какая проблема решается.

## Входные данные
- Файлы, которые нужно прочитать
- Ссылки на тесты, которые должны пройти

## Критерии готовности
- [ ] Критерий 1
- [ ] Критерий 2
- [ ] Тесты проходят: `ctest --output-on-failure`

## Ограничения
- Что нельзя менять (например: формат циклограммы, маппинг каналов)
- Затронутые модули
```

## Приоритеты

Используй префикс в имени файла:
- `P1-TASK-NNN-...` — критично, делать первым
- `P2-TASK-NNN-...` — обычный приоритет
- `P3-TASK-NNN-...` — улучшение, можно отложить

## Статусы

После обработки задачи AI переименовывает или добавляет статус:
- `TASK-NNN-...` → в очереди
- `DONE-TASK-NNN-...` → выполнено
- `BLOCKED-TASK-NNN-...` → заблокировано, нужна помощь
