## Что за приложение

**pyro_stand** — десктоп-приложение (Windows, 1280×800) для контроля пуска ракеты.  
Оператор загружает циклограмму, запускает отсчёт, следит за срабатыванием пиропатронов.

### Фазы и цвет таймера
- `Idle` / `Loaded` → `#8b949e` серый
- `Countdown` → `#e3b341` жёлтый, мигает
- `Running` → `#3fb950` зелёный
- `Stopped` → `#f85149` красный
- `Completed` → `#3fb950` зелёный

### Таблица событий — ровно 4 строки, 7 колонок
`# | СОБЫТИЕ | КАНАЛЫ | ПЛАН МС | ФАКТ МС | ОТКЛ МС | СТАТУС`

Данные фиксированы в коде — всегда 4 события (4 группы пиропатронов).
- Pending-строка: фон `#1c2d3a`
- ОТКЛ МС цвет: ≤5 мс → `#3fb950`, 6–50 → `#e3b341`, >50 → `#f85149`
- СТАТУС: «ожид.» (серый) / «OK» (зелёный) / «FAIL» (красный)

### 🔴 ПРИОРИТЕТ 1 — T18: Блок прогресса (вместо таймлайна)

**Решение TL (от 2026-06-20):** таймлайн убрать, заменить блоком прогресса.

Текущий таймлайн (52px горизонтальная полоса) не работает с реальными данными —

2. «До следующего события: 3.2 с» — заметнее чем сейчас
3. 8 точек каналов — уже есть, сохрани

Файл: `design/mockups/T18-Progress-v1.html`

### 📋 ПРИОРИТЕТ 2 — Панель управления (группировка)
Кнопки выглядят как плоский список. Нужна карточка с группами:
- **«Подготовка»**: ЗАГРУЗИТЬ НА БОРТ + УСТАНОВИТЬ ВРЕМЯ
- **«Управление»**: СТОП + СБРОС
- COM/БЦВМ индикаторы — крупнее и заметнее (сейчас мелкие)

### 📋 ПРИОРИТЕТ 3 — Лог-панель
Текущий лог — `QTextEdit` без структуры. Хочется 2-колоночный вид:
- Колонка 1: `HH:MM:SS` (фиксированная ширина, `#8b949e`)
- Колонка 2: текст сообщения с цветом по типу:
  - `system` → `#8b949e`
  - `event` (OK) → `#3fb950`, фон `rgba(63,185,80,0.07)`
  - `event-post` (FAIL) → `#f85149`, фон `rgba(248,81,73,0.07)`

## Как создать макет

1. Создай файл `design/mockups/T{номер}-{название}.html`
2. Первые строки файла:
```html
<!-- STATUS: draft -->
<!-- TASK: T18 — Progress block -->
<!-- FROM: Design AI -->
<!-- DATE: дата -->
```
3. Размер окна в макете: 1280×800px
4. Используй только цвета из таблицы выше
5. Внизу — короткий комментарий: как реализуется в Qt (виджет/метод)

**Не читай другие файлы репозитория — всё что нужно есть в этом файле.**
