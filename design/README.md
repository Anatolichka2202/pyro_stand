# Дизайн-проект pyro_stand

## О проекте

Дизайн тёмной темы для десктопного приложения контроля пуска ракеты.
Целевой экран: 1280×800, Windows-десктоп.

Референсная спецификация UI — [`spec.md`](spec.md).

---

## Что было реализовано

### Таймер (mainwindow.cpp → `updateTimer`)

Формат отображения без префикса "T-":
- До старта: `мм:сс` (< 1 ч) или `чч:мм:сс` (≥ 1 ч), жёлтый `#e3b341`
- В полёте: `мм:сс.д` (десятые), зелёный `#3fb950`
- СТОП: фиксированный текст `СТОП`, красный `#f85149`
- Шрифт: JetBrains Mono, 52px, weight 600

### Лог-панель (mainwindow.cpp → `addLog`)

Структурированные строки через `QTextEdit` с HTML:
```
▌  HH:mm:ss  УРОВЕНЬ  сообщение
```
- Цветная полоса `▌` (U+258C) вместо CSS `border-left` (Qt не поддерживает)
- Уровни: `INFO` (серый) / `СОБЫТ.` (зелёный/красный) / `ОШИБКА` (красный)

### Таблица событий (mainwindow.cpp → `updateTable`, `updateTableRow`)

7 колонок: `# | СОБЫТИЕ | КАНАЛЫ | ПЛАН МС | ФАКТ (ТИК) | ОТКЛ МС | СТАТУС`
- Заголовок кол. 4 меняется: `ФАКТ (ТИК)` во время полёта → `ФАКТ МС` после анализа
- Текущая строка: синяя полоска в кол. 0 (`#58a6ff`), подсветка строки `#1c2d3a`
- Высота таблицы capped: max 12 строк до скролла
- Цвет отклонения: ≤5 мс → зелёный / 6–50 → жёлтый / >50 → красный

### Таймлайн (timeline_widget.cpp)

Горизонтальная полоса 72px:
- Круги событий R=5 (обычное) / R=7 (T0)
- Кластеризация: если события ближе 14px — объединяются в один узел с числом
- Playhead с glow-эффектом (3 слоя)
- Маркер T0 отдельным цветом

### Индикаторы COM / БЦВМ (mainwindow.cpp)

Чип-виджеты (QFrame) с цветной точкой и подписью статуса.
Зелёный при соединении, серый при отсутствии.

### Фазы и состояния

| Фаза | Цвет таймера | Заголовок |
|------|-------------|-----------|
| Idle / Loaded | `#8b949e` | ДО ЗАПУСКА |
| Countdown | `#e3b341` (мигает) | ДО ЗАПУСКА |
| Running | `#3fb950` | ВРЕМЯ В ПОЛЁТЕ |
| Completed | `#3fb950` | ВРЕМЯ В ПОЛЁТЕ |
| Stopped | `#f85149` | СТОП |

## Решения и компромиссы

**Таймлайн оставлен** (не заменён блоком прогресса).
Кластеризация решила проблему схлопывания 4 событий в 70мс-окне:
узлы показывают счётчик событий при перекрытии.

**`border-left` в QTextEdit не работает** — Qt HTML-рендерер не поддерживает
CSS `border-left` на inline-элементах. Заменено символом `▌` (U+258C LEFT HALF BLOCK).

**Таблица не растягивается на весь экран** — `setMaximumHeight` по числу строк.
