# Руководство оператора — Пиростенд

Версия приложения: v1.0.0  
Платформа: Windows 10 / Linux (Ubuntu 22.04+)

---

## 1. Назначение

Приложение контролирует срабатывание пиротехнических устройств во время огневого испытания ракетного двигателя. Оно сравнивает фактические времена срабатывания каналов с плановой циклограммой и выдаёт протокол отклонений.

---

## 2. Аппаратные требования

| Компонент | Описание |
|-----------|----------|
| **Датчик COM** | Устройство измерения тока в пиролиниях. Генерирует 1 байт/мс, 115200 Бод. Windows: `COM7`, Linux: `/dev/ttyUSB0` |
| **БЦВМ** | Бортовой компьютер. IP: `192.168.17.246`, принимает циклограмму по TFTP (порт 69) или UDP (порт 4000) |
| **Сеть** | PC должен быть в одной подсети с БЦВМ |

---

## 3. Запуск приложения

```
pyro_stand.exe                    # Windows
./pyro_stand                      # Linux
```

При старте приложение:
- Открывает COM-порт (индикатор **COM** слева вверху)
- Ждёт готовности оператора

Если COM-порт недоступен — кнопка **ЗАГРУЗИТЬ НА БОРТ** неактивна. Проверьте подключение датчика и перезапустите приложение.

---

## 4. Подготовка к испытанию

### 4.1 Файл циклограммы

Разместите файл `cyclogram.ini` в той же папке, что и исполняемый файл. Формат:

```ini
SET_UTC_TIME = 10:00:00           # время часов лаборатории (UTC)
START_UTC_TIME = 10:00:30         # плановое время старта (UTC)
IGNITE_PYRO_CANDLES_ENGINES_9_TO_12 = -8000   # -8 сек до T0
FIRE_STARTING_CHAMBER_ENGINES_9_TO_12 = -7950 # -7.95 сек до T0
LIFT_OFF_CONTACT = 0              # T0 = контакт подъёма
CLOSE_MAIN_VALVES_ENGINES_9_TO_12 = 20000     # +20 сек от T0
```

Ключи событий должны совпадать с маппингом каналов (см. раздел 7).

### 4.2 Установка времени старта

В поле **СТАРТ** введите актуальное плановое время в формате `ЧЧ:ММ:СС`. Нажмите **Установить**. Время должно быть позже `SET_UTC_TIME`.

### 4.3 Выбор протокола передачи

Переключатель **TFTP / UDP** (по умолчанию — TFTP):
- **TFTP** — надёжная передача с подтверждением доставки. **Рекомендуется.**
- **UDP** — без подтверждения. Применять только если БЦВМ не поддерживает TFTP.

---

## 5. Порядок проведения испытания

1. Убедитесь, что индикаторы **COM** и **БЦВМ** зелёные
2. Нажмите **▤ ЗАГРУЗИТЬ НА БОРТ**
   - Статус под кнопкой: *Проверка БЦВМ...* → *Загрузка циклограммы...* → *Циклограмма загружена ✓*
3. Начнётся обратный отсчёт до старта (фаза **ОБРАТНЫЙ ОТСЧЁТ**)
4. В момент T0 (контакт подъёма) таймер переключается на **ВРЕМЯ В ПОЛЁТЕ**
5. По завершению циклограммы фаза меняется на **ЗАВЕРШЕНО**
6. Таблица результатов заполняется автоматически

**Аварийная остановка**: кнопка **⏹ СТОП** отправляет команду `STOP` на БЦВМ и останавливает запись.

---

## 6. Интерпретация результатов

### 6.1 Таблица событий

| Колонка | Описание |
|---------|----------|
| **#** | Порядковый номер события |
| **СОБЫТИЕ** | Описание из циклограммы |
| **КАНАЛЫ** | Отслеживаемые каналы (1–8). `Δ1мс` = разброс внутри группы |
| **ПЛАН МС** | Плановое время от T0 (мс) |
| **ФАКТ МС** | Фактическое время от T0 (мс) |
| **ОТКЛ МС** | Отклонение от плана |
| **СТАТУС** | Итог срабатывания |

Под многоканальными событиями появляются строки **↳ Канал N** с индивидуальным временем каждого канала. Последний сработавший канал помечается **← послед.**

### 6.2 Статусы

| Статус | Цвет | Значение |
|--------|------|----------|
| **ОК** | Зелёный | Отклонение 0 мс |
| **ОК (Nмс)** | Зелёный | Отклонение 1–4 мс — в норме |
| **+N мс** | Жёлтый | Отклонение ≥ 5 мс — информативно |
| **КП ШТАТНО** | Зелёный | Контакт подъёма в пределах ±1 с от плана |
| **КП ОТКЛ** | Жёлтый | Контакт подъёма опоздал/опередил более 1 с |
| **НЕ СРАБОТАЛО** | Красный | Канал не зафиксирован в потоке |

> Все отклонения считаются **относительно фактического T0** (момента контакта подъёма), а не планового. Это исключает влияние погрешности синхронизации БЦВМ на результаты анализа.

### 6.3 Сводная строка

Пример: `Результат: 4 ✓ / 0 ≈ / 0 ✗  ·  Макс. откл.: 1 мс`

- **✓** — точные срабатывания (0 мс)
- **≈** — срабатывания с ненулевым отклонением
- **✗** — несработавшие каналы

---

## 7. Маппинг каналов

| Событие в циклограмме | Маска | Каналы |
|-----------------------|-------|--------|
| `IGNITE_PYRO_CANDLES_ENGINES_9_TO_12` | `0x03` | 1, 2 |
| `FIRE_STARTING_CHAMBER_ENGINES_9_TO_12` | `0x1C` | 3, 4, 5 |
| `CLOSE_MAIN_VALVES_ENGINES_9_TO_12` | `0x60` | 6, 7 |
| `LIFT_OFF_CONTACT` | `0x80` | 8 (синхроимпульс T0) |

События, ключ которых не совпадает ни с одной строкой маппинга, отображаются в таблице без статуса и не отслеживаются.

---

## 8. Повторный запуск

После завершения испытания нажмите **↺ СБРОС**. Приложение перезагружает циклограмму и возвращается в исходное состояние. Лог-файл предыдущей сессии сохраняется.

---

## 9. Экспорт результатов

Кнопка **CSV** в нижней части экрана сохраняет таблицу результатов в `.csv` файл для дальнейшего анализа.

---

## 10. Лог-файл сессии

Каждая сессия создаёт файл:
```
logs/session_YYYY-MM-DD_HH-MM-SS/pyro_stand.log
```

Формат строки: `HH:mm:ss.zzz | LEVEL | сообщение`

Временны́е метки в лог-файле — **не системное время**, а расчётное время от `SET_UTC_TIME` плюс аппаратный счётчик байт датчика. Это обеспечивает тик-точную привязку без зависимости от NTP.

---

## 11. Индикаторы состояния

| Индикатор | Зелёный | Красный |
|-----------|---------|---------|
| **COM** | Порт открыт | Порт недоступен или ошибка |
| **БЦВМ** | Отвечает на ping | Недоступна |
| **Каналы 1–8** | Канал сработал | — |

---

## 12. Частые проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| Кнопка ЗАГРУЗИТЬ неактивна | COM-порт не открыт | Проверьте подключение датчика, перезапустите |
| "БЦВМ недоступна" | Нет сети или неверный IP | Проверьте сетевое соединение с `192.168.17.246` |
| "Ошибка загрузки" под кнопкой | TFTP таймаут | Проверьте БЦВМ, попробуйте UDP |
| Все события НЕ СРАБОТАЛО | COM читает нули | Проверьте датчик |
| КП ОТКЛ | T0 опоздал/опередил более 1 с | Информативно, анализ корректен |
