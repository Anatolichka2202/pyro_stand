## 2. Контракт данных / что нужно виджетам от C++

### 2.1 Таймер (`m_timerLabel` + caption)
- Caption по фазе: `Idle/Loaded/Countdown` → **«ДО ЗАПУСКА»**, `Running/Completed` → **«ВРЕМЯ В ПОЛЁТЕ»**.
- Значение: формат `чч:мм:сс` (боевая циклограмма ~30 мин → нужны минуты/часы), **без префикса «T-»**, без знака.
- Цвета по фазе — без изменений (`#484f58 / #e3b341 / #3fb950 / #f85149`).

### 2.2 Каналы (новый рендер)
- Текст чипа: `"Канал %1".arg(n)` вместо `"CH%1"`.
- Состояния (цвет точки + рамки): `idle #484f58`, `ожидает #e3b341` (+ пульс), `ok #3fb950`, `fail #f85149`.
- Привязка: чип красится, когда событие с этим каналом меняет статус (логика `updateChannelDot` уже есть — меняется только разметка/подпись).

### 2.3 Таймлайн (`TimelineWidget`)
Существующее API сохраняется: `setEvents() · setPlayheadMs() · markEventFired(id, "ok"/"fail") · reset()`.
**Новое — кластеризация:**
- При построке группировать события, чьи x-координаты ближе **14 px** (диаметр кружка + зазор), в один узел.
- Статус узла = худший среди членов (`fail > tolerance > ok > pending`).
- Подпись узла = число членов; диапазон номеров под осью («1–4»).
- Клик по узлу → поповер со списком членов (номер · описание · отклонение). Это новый виджет-оверлей.
- T0: добавить подпись «T0» над красной линией (сейчас её нет).
- Playhead: добавить `box-shadow` glow `rgba(88,166,255,0.5)`, анимировать позицию между тиками во время `Running`.
- Высота: **72 px** (было 52/64) — иначе номер и время наезжают на ось.

### 2.4 Таблица (7 колонок — без изменений в структуре)
- Колонка «Каналы»: значение — **`"Канал N"` через запятую** (`Канал 1, Канал 2`). Ширина ~190 px.
- Колонка «Событие» — `description` (fallback `key`), `ellipsis` при переполнении.
- Пороги статуса (как в коде): `dev≤0` зелёный «✓ ОК» · `dev≤5` жёлтый «✓ ОК (±N мс)» · `dev>5` красный «✗ НЕ ОК (±N мс)» · `fail` красный «✗ НЕ СРАБОТАЛО».
- Текущая/следующая строка: подсветка `#1c2d3a` + левая полоса `#58a6ff` (есть, усилить полосой).

#### 2.4.1 ⚠️ Два режима таблицы (по уточнению Анатолия) — см. `Scenario-v1.dc.html`
Таблица меняет смысл колонок по фазе — это нужно учесть в `updateTableRow`:

**A. Во время полёта (`Running`) — абсолютные тики:**
- При «Загрузить на борт» начинается чтение тиков внешнего таймера (COM-порт).
- При срабатывании показываем **абсолютный индекс тика** в колонке «Факт» (заголовок меняем на **«Факт (тик)»**).
- «План мс» и «Откл мс» = **«—»** (пересчёт ещё невозможен — нет привязки к T0).
- «Статус» = только **«✓ сработало»** / pending. Без допусков.

**B. После отработки (`Completed`) — относительное время + анализ:**
- **Канал 8** (событие «Контакт подъёма», `time_ms = 0`) = **относительный ноль (T0)**.
  По его абсолютному тику пересчитываем все остальные: `rel_ms(e) = tick(e) − tick(Канал8)` (в мс).
- Тогда заполняем «Факт мс» (относительное), «Откл мс» = `rel_ms − plan_ms`, «Статус» с допусками.
- Заголовок «Факт» возвращается к **«Факт мс»**.

**C. Канал/событие не сработал вообще:**
- «Факт мс» и «Откл мс» = **`NA`** (красный), «Статус» = **«✗ не сработало»** (красный).
- Чип канала в панели каналов → красный.

> Вопрос: если **Канал 8 сам не сработал** — пересчёт невозможен. Что показываем? Предлагаю:
> весь блок анализа в режиме «нет T0», все строки `NA`, явная ошибка в логе. Подтверди.

### 2.5 Лог
- Тип сигнала → метка уровня: `system→INFO (синий)`, `event→СОБЫТ. (зелёный)`, `event-post→ОШИБКА (красный)`, `error→ОШИБКА (красный)`. Допуск → жёлтый «СОБЫТ.».
- Разметка строки: `[время 78px моно] [уровень 54px] [сообщение flex]` + левая полоса 3px цвета уровня.
- Слабая подложка строки для warn/error (`rgba(...,0.05)`).
