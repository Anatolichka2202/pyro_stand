# design/mockups — макеты редизайна Pyro Stand

Индекс папки. Все `.dc.html` открываются прямо в браузере (рядом нужен `support.js`).

## Документация
- **`DESIGN-DOC.md`** — главный документ: UX-решения, контракт данных/API по каждому
  виджету, открытые вопросы к Артёму. Начинать отсюда.

## Макеты
| Файл | Что показывает |
|------|----------------|
| `Scenario-v1.dc.html` | **Полный путь оператора** — 5 шагов с переходами: Ожидание → Загружено → Отсчёт → Выполнение (абс. тики) → Завершено (пересчёт по Каналу 8 + анализ + NA) |
| `FullScreen-v1.dc.html` | Собранный экран целиком, фаза «Выполнение» — эталон компоновки |
| `T18-Timeline-v3.dc.html` | Таймлайн: кружки, T0, playhead с glow, групп-узел для слипающихся кластеров (4 состояния) |
| `T19-ControlPanel-v1.dc.html` | Панель управления: группы кнопок, COM/БЦВМ плашки (2 фазы) |
| `T20-LogPanel-v1.dc.html` | Лог: колонки время · уровень · сообщение + цветная полоса |

## Прочее
- `support.js` — рантайм для рендера `.dc.html` (не редактировать).
- Автономная версия сценария (один файл, работает офлайн) лежит в корне проекта:
  `Сценарий - путь оператора.html`.
